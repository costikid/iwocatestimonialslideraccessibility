Case Study: Making a Testimonial Slider Keyboard-Accessible

⚠ The Problem: Slider with No Keyboard Support for right and left arrows

The original widget was built on the Slick slider jQuery library. The control buttons were plain <div> elements — not real <button>s — with tabindex="0" on every item. This created several accessibility barriers:

  • No ARIA tab roles (role="tablist", role="tab", role="tabpanel")
  • No aria-selected or aria-controls attributes
  • No keydown handler for arrow keys
  • Screen reader users could not identify which slide was active
  • Assistive technology users could not navigate the slider with right and left arrow keys

In the recording above I keep pressing the right and left arrow keys to move between the different slides using the buttons, but the slider does not respond — the original widget has no keyboard handler, so arrow-key navigation is impossible.

✓ The Solution

I implemented the WAI-ARIA Tabs pattern to bring the slider up to WCAG accessibility standards. The key changes enable full keyboard navigation with a roving tabindex so screen reader users can move between slides using arrow keys alone:

After — HTML (active tab enters Tab order) 
<!-- Active tab: tabindex="0" (in Tab order) -->
<button role="tab"
        aria-selected="true"
        tabindex="0">
  Marketing
</button>

<!-- Inactive tabs: tabindex="-1" (skipped by Tab, -->
<!-- reachable via arrow keys) -->
<button role="tab"
        aria-selected="false"
        tabindex="-1">
  Purchasing equipment
</button>
After — JavaScript (roving tabindex in activate()) 
function activate(index, moveFocus) {
  const newIndex = (index + tabs.length) % tabs.length;

  tabs.forEach((tab, i) => {
    const selected = i === newIndex;
    tab.setAttribute('aria-selected', String(selected));
    tab.tabIndex = selected ? 0 : -1;  // roving tabindex
    tab.classList.toggle('topic-tab--active', selected);
  });

  panels.forEach((panel, i) => {
    panel.hidden = i !== newIndex;
  });

  if (moveFocus) tabs[newIndex].focus();
  movePointer(tabs[newIndex]);
}
After — JavaScript (arrow key handler) 
tablist.addEventListener('keydown', (e) => {
  const currentIndex = getActiveIndex();
  let newIndex = null;

  switch (e.key) {
    case 'ArrowRight':
    case 'ArrowDown':
      newIndex = currentIndex + 1;
      break;
    case 'ArrowLeft':
    case 'ArrowUp':
      newIndex = currentIndex - 1;
      break;
    case 'Home':
      newIndex = 0;
      break;
    case 'End':
      newIndex = tabs.length - 1;
      break;
    default:
      return;
  }

  e.preventDefault();
  activate(newIndex, true);
});
